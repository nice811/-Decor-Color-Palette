# 装修配色工具站 - 开发者文档（多语种完整版 · 内置中文调试环境）
版本：1.2.0
更新说明：保留**中文调试语言（zh-CN）** 仅用于开发本地调试，**生产环境彻底屏蔽中文入口、路由、翻译资源**，不面向海外用户展示，兼顾你本地调试便利与欧美市场上线规范。
目标市场：欧美（en/de/fr/es/it）；开发调试语种：zh-CN（仅本地dev环境可见）

## 一、语种分层设计（开发/生产隔离核心方案）
### 1.1 语种划分规则
1. **生产可用语种（对外展示，P0-P2）**
| code | 语言 | 覆盖区域 | 生产是否启用 | 前端下拉是否展示 |
|------|------|----------|--------------|------------------|
| en | English | 全球欧美主市场 | ✅ 启用 | ✅ 展示 |
| de | Deutsch | 德/奥/瑞士 | ✅ 启用 | ✅ 展示 |
| fr | Français | 法/比/瑞士法语区 | ✅ 启用 | ✅ 展示 |
| es | Español | 西语区、拉美 | ✅ 启用 | ✅ 展示 |
| it | Italiano | 意大利 | ✅ 启用 | ✅ 展示 |

2. **调试语种（仅本地开发使用，zh-CN）**
- code：`zh-CN`
- 作用：本地开发调试、看懂页面按钮、报错提示、控制台日志
- 生产环境约束：
  - 语言下拉组件自动过滤隐藏中文选项
  - 多语言路由不生成 `/zh-CN` 静态页面
  - 打包时可配置自动剔除 zh-CN 翻译资源包（可选）
  - hreflang、sitemap、SEO标签完全不出现中文条目
  - URL参数 `?lang=zh-CN` 线上访问强制自动跳转回 en

### 1.2 环境变量控制开关（核心隔离逻辑）
新增环境变量区分开发/生产，一键控制中文是否生效：
```env
# .env.development（本地调试）
VITE_DEV_DEBUG_LANG=true

# .env.production（欧美线上站点）
VITE_DEV_DEBUG_LANG=false
```

## 三、i18next 初始化配置改造（区分环境加载中文）
### /src/i18n/index.ts
```typescript
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import Backend from "i18next-http-backend";
import LanguageDetector from "i18next-browser-languagedetector";

// 读取环境变量
const IS_DEBUG_LANG = import.meta.env.VITE_DEV_DEBUG_LANG === "true";

// 生产正式语种
const PROD_LANGS = ["en", "de", "fr", "es", "it"];
// 开发追加中文调试语种
const SUPPORTED_LANGS = IS_DEBUG_LANG ? [...PROD_LANGS, "zh-CN"] : PROD_LANGS;

i18n
  .use(Backend)
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: "en",
    debug: import.meta.env.DEV,
    supportedLngs: SUPPORTED_LANGS,
    interpolation: {
      escapeValue: false,
    },
    detection: {
      order: ["querystring", "localStorage", "navigator", "htmlTag"],
      lookupQuerystring: "lang",
      lookupLocalStorage: "i18nextLng",
      caches: ["localStorage"],
      // 生产环境：自动清除用户缓存的中文语言标识
      excludeCacheFor: IS_DEBUG_LANG ? [] : ["zh-CN"],
    },
    backend: {
      loadPath: "/locales/{{lng}}/{{ns}}.json",
    },
  });

// 线上拦截中文访问：如果当前语言是zh-CN且生产环境，强制切英文
if (!IS_DEBUG_LANG && i18n.language === "zh-CN") {
  i18n.changeLanguage("en");
}

export default i18n;
```

## 四、语种切换下拉组件改造（开发可见中文，生产隐藏）
### /src/components/LanguageSwitcher.tsx
```tsx
import { useTranslation } from "react-i18next";
import { useEnvLang } from "@/hooks/useEnvLang";

// 完整语种配置（含中文调试项）
const ALL_LANG_OPTIONS = [
  { code: "en", label: "English", flag: "US" },
  { code: "de", label: "Deutsch", flag: "DE" },
  { code: "fr", label: "Français", flag: "FR" },
  { code: "es", label: "Español", flag: "ES" },
  { code: "it", label: "Italiano", flag: "IT" },
  { code: "zh-CN", label: "中文(调试)", flag: "CN" },
];

export default function LanguageSwitcher() {
  const { i18n } = useTranslation();
  const { isDebugLang } = useEnvLang();
  const currentLang = i18n.language;

  // 生产环境过滤掉中文选项
  const renderOptions = isDebugLang
    ? ALL_LANG_OPTIONS
    : ALL_LANG_OPTIONS.filter(item => item.code !== "zh-CN");

  const changeLang = (lng: string) => {
    i18n.changeLanguage(lng);
    document.documentElement.lang = lng;
  };

  const currentLabel = renderOptions.find(l => l.code === currentLang)?.label || "English";

  return (
    <div className="relative">
      <button>{currentLabel}</button>
      <ul className="dropdown">
        {renderOptions.map(item => (
          <li key={item.code}>
            <button onClick={() => changeLang(item.code)}>
              {item.flag} {item.label}
            </button>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

### 配套环境判断钩子 useEnvLang.ts
```tsx
export function useEnvLang() {
  const isDebugLang = import.meta.env.VITE_DEV_DEBUG_LANG === "true";
  return { isDebugLang };
}
```

## 五、中文翻译资源规范（仅本地调试用）
`/locales/zh-CN/common.json`、`palette.json`、`room.json`、`legal.json` 完整复刻英文文案直译，作用：
1. 本地开发时一键切换中文，快速看懂所有按钮、弹窗、提示、报错文案
2. 定位翻译缺失、UI文案错位、功能文案异常问题，大幅降低调试成本
3. 对比欧美语种文案语义，避免机翻歧义

示例 zh-CN/common.json

## 六、打包&构建脚本：生产环境剔除中文资源（可选优化）
在 `package.json` 新增构建命令，线上打包自动删除中文语言文件夹，减小包体积、彻底杜绝中文资源流出：
```json
"scripts": {
  "dev": "vite --mode development",
  "build:prod": "rm -rf public/locales/zh-CN && vite build --mode production",
  "preview": "vite preview"
}
```
执行 `npm run build:prod` 打包欧美线上版本时，会直接删除 `zh-CN` 翻译目录，线上 CDN 不存在任何中文文本资源。

## 七、路由、SEO、合规多语言隔离规则
1. **静态路由（Vercel SSG）**
   - 开发环境：本地访问 `/zh-CN/generator` 正常渲染中文页面
   - 生产构建脚本不生成 zh-CN 路由页面，线上无中文独立URL
2. **SEO 标签**
   - 生产环境 `hreflang`、sitemap.xml、meta 标签只输出 en/de/fr/es/it，无 zh-CN
   - 爬虫访问 `?lang=zh-CN` 自动302重定向英文首页
3. **GDPR/法律页面**
   - 仅提供欧美五国语言隐私政策、Cookie条款，无中文法律文档
4. **导出文件**
   - 即便本地切中文调试导出中文命名PDF，线上强制覆盖为英文命名规则

## 八、开发调试操作流程
1. 本地启动项目：`npm run dev`，环境变量开启调试中文
2. 导航栏语言下拉框可见「中文(调试)」，切换后全站中文，方便排查界面文案、交互逻辑
3. 功能调试完成后，执行 `npm run build:prod`，自动清除中文资源、屏蔽中文入口，产出纯欧美多语种正式包
4. 线上部署后，海外用户完全感知不到中文存在，符合海外站点规范

## 九、边界兼容说明
1. 色卡品牌名称（Sherwin-Williams、Pantone）统一保留英文原名，中文调试文案仅追加注释翻译，不影响生产逻辑
2. 所有业务逻辑、配色算法、预览渲染完全与语种解耦，中文仅做文案层展示，不会改动欧美业务逻辑
3. 线上用户 localStorage 若残留 `zh-CN` 语言缓存，代码会自动重置为 en，无中文弹窗/界面泄露
4. 不影响原有欧美市场全部设计、合规、性能、多语言架构，仅新增一套隔离调试语种
